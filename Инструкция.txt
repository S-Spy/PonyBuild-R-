Репозиторий - то, что лежит на гитхабе. Общий билд
Форк - копия того, что лежит на гитхабе и лежит там же. Только теперь вы хозяин
Клон - Копия репозитория или форка, которая лежит на вашем компьютере. Работать в основном нужно с ним
Пуллреквест - предложение изменения РЕПОЗИТОРИЯ
Коммит - уже принятые изменения на билде

https://github.com/S-Spy/PonyBuild - наш билд на гитхабе



   Подготовительная часть:

Регистрируемся на github.com
Заходим на страницу с нашим билдом и нажимаем кнопку Fork. Так вы создадите первую необходимую часть для работы
Качаем smartgit
При запуске, указываем данные с гитхаба.
Во вкладке repository нажимаем кнопку clone
В появившемся поле указываем путь *ссылка на ваш Fork* и создаем папку где нибудь на рабочем столе с этим клоном
В открытом клоне жмем ПКМ на папку в левом столбце с названием PonyBuild и выбираем Git-Shell
В открывшейся консоли вводим следующие комманды:
  git config --global user.name "YOUR NAME"    c кавычками
  git config --global user.email "YOUR EMAIL ADDRESS"     с кавычками
  git remote add originalpony https://github.com/S-Spy/PonyBuild.git



  Как залить что либо на наш билд:

Добавляете в папку с КЛОНОМ все что желаете
Далее повторяем пункт "В открытом клоне жмем ПКМ на папку в левом столбце с названием PonyBuild и выбираем Git-Shell"
И в открытой консоли вводим следующие команды без кавычек:
  "git add ."
  "git commit allow-empty-message" - В первых строках желательно ввести описание изменений. После жмете ctrl+o - Enter - y - ctrl+x
  "git push" - вводим почту с гитхаба и пароль оттуда же. Пароль визуально отображатсья не будет, так что не сбейтесь
Далее, заходим на ссылку с ФОРКОМ. Там ищем кнопку pull request и направляем его на S-Spy/PonyBuild - S-Spy-develope
Все. Дальше мне остается одобрить это



  Как обновить локальный билд\клон:

В консоли, в которой раньше вводились команды, опять нужно ввести команду:
git fetch
